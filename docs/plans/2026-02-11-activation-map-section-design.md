# ActivationMapSection Design

## Overview
A pure presentation component that renders activation stages as a horizontally scrolling row of cards with transition connectors between them. Each stage shows signal strength, trigger events, value moments, and drop-off risk. The primary activation level is visually highlighted.

## Problem Statement
Product profile pages need to display the activation map generated by the AI analysis pipeline. Users need to see the progression of activation stages, understand what triggers movement between stages, and identify the primary activation level at a glance.

## Expert Perspectives

### Technical
- Horizontal layout matches the mental model of progression (left-to-right flow)
- Keep normalizeDropOffRisk in-file until the pattern repeats elsewhere
- Use existing Card and Badge components from the UI library
- Pure presentation — no Convex queries, receives data as props

### Simplification Review
- No extracted sub-components (StageCard, TransitionConnector) — inline everything in the map loop
- Replace transition lookup Map with simple `.find()` — only 3-5 stages, clarity wins
- Keep color maps as simple lookup objects, not over-abstracted
- Single file, ~80-100 lines total

## Proposed Solution

### File: `src/components/product-profile/ActivationMapSection.tsx`

**Props:**
```typescript
interface ActivationMapSectionProps {
  activationMap: ActivationMap | null | undefined;
}
```

**Structure:**
1. Empty state guard: if null/undefined/empty stages, render "No activation map available"
2. Sort stages by `level` ascending
3. Render `flex items-center gap-2 overflow-x-auto pb-2` row
4. For each stage, render a Card (min-w-[240px] shrink-0):
   - CardHeader: "Level N: Name" + signal strength Badge
   - CardContent: trigger events list, value moments list, drop-off risk Badge
   - Primary stage: `ring-2 ring-indigo-500 bg-indigo-50`
5. Between cards: inline SVG arrow connector with transition metadata (trigger events, timeframe)

**Helpers (all in-file):**
- `normalizeDropOffRisk(risk: unknown)` → `"low" | "medium" | "high"` — defensive for both string and `{level, reason}` shapes (required by acceptance criteria)
- Signal strength colors: `{ very_strong: indigo, strong: green, medium: amber, weak: gray }`
- Risk colors: `{ low: green, medium: amber, high: red }`
- Transition lookup: `.find()` on transitions array matching from_level/to_level

**No extracted sub-components.** Everything inlined in the single component function.

### Test File: `src/components/product-profile/ActivationMapSection.test.tsx`

8 tests, one per acceptance criterion:
1. Renders stages sorted by level in horizontal layout
2. Each stage shows level + name and signal strength badge with correct color
3. Trigger events and value moments render on each stage
4. Drop-off risk badge renders green/amber/red
5. Transition connectors show trigger events and timeframe
6. Primary activation level gets ring-2 ring-indigo-500 highlight
7. Handles drop_off_risk as `{ level, reason }` object shape
8. Empty state renders when activationMap is null/undefined/empty

## Alternatives Considered

1. **Vertical timeline layout** (technical architect suggestion) — Rejected because acceptance criteria explicitly require "horizontal progression cards" and hints specify "Flex row with overflow-x-auto"
2. **Extracted sub-components** (StageCard, TransitionConnector) — Rejected per simplification review; each is only ~15-20 lines of JSX, extraction adds overhead without benefit
3. **Shared utils file for normalizeDropOffRisk** — Deferred; keep in-file until the pattern repeats in other components

## Implementation Plan

### Step 1: Create `src/components/product-profile/ActivationMapSection.tsx`

Single file, pure presentation component (~80-100 lines).

**Imports:**
```typescript
import type { ActivationMap, StageTransition } from "../../../convex/analysis/outputs/types";
import { Card, CardHeader, CardTitle, CardContent } from "../ui/card";
import { Badge } from "../ui/badge";
```

**Color maps (module-level constants):**
```typescript
const SIGNAL_COLORS: Record<string, string> = {
  very_strong: "bg-indigo-100 text-indigo-800",
  strong: "bg-green-100 text-green-800",
  medium: "bg-amber-100 text-amber-800",
  weak: "bg-gray-100 text-gray-800",
};

const RISK_COLORS: Record<string, string> = {
  low: "bg-green-100 text-green-800",
  medium: "bg-amber-100 text-amber-800",
  high: "bg-red-100 text-red-800",
};
```

**Helper (in-file):**
```typescript
function normalizeDropOffRisk(risk: unknown): "low" | "medium" | "high" {
  if (typeof risk === "string") return risk as "low" | "medium" | "high";
  if (risk && typeof risk === "object" && "level" in risk) {
    return (risk as { level: string }).level as "low" | "medium" | "high";
  }
  return "medium"; // safe default
}
```

**Component logic:**
1. Guard: if `!activationMap || !activationMap.stages.length` → render empty state `<p>No activation map available</p>`
2. `const sortedStages = [...activationMap.stages].sort((a, b) => a.level - b.level)`
3. Render `<div className="flex items-center gap-2 overflow-x-auto pb-2">` containing:
   - For each stage (with `index`):
     - If `index > 0`, render transition connector: find matching transition via `activationMap.transitions.find(t => t.from_level === sortedStages[index-1].level && t.to_level === stage.level)`. Render an SVG arrow (→ character or simple div) with trigger events and timeframe text.
     - Render `<Card>` with `min-w-[240px] shrink-0`. If `stage.level === activationMap.primary_activation_level`, add `ring-2 ring-indigo-500 bg-indigo-50`.
     - `<CardHeader>`: title "Level {level}: {name}" + `<Badge>` with signal strength text and color from `SIGNAL_COLORS`.
     - `<CardContent>`:
       - Trigger events: `<ul>` list
       - Value moments unlocked: `<ul>` list
       - Drop-off risk: `<Badge>` with normalized risk and color from `RISK_COLORS`
