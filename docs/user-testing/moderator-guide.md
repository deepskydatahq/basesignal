# User Test Moderator Guide

Complete session flow guide for conducting H2: Interview Completion user tests.

## Session Overview

| Aspect | Details |
|--------|---------|
| Duration | ~20 minutes total |
| Format | Live moderated via Zoom |
| Recording | Cloud recording with transcript |
| Focus | Interview completion flow |

## Pre-Session Setup (5 min before)

- [ ] Zoom meeting created with recording enabled
- [ ] Basesignal app accessible and tested
- [ ] Consent script ready (see `consent-script.md`)
- [ ] Note-taking document open
- [ ] Timer ready for 15-minute interview portion
- [ ] Quiet environment, headphones on

### Zoom Setup

1. Start meeting 5 minutes early
2. Verify cloud recording is enabled
3. Have Basesignal URL ready to paste in chat
4. Open consent script in separate window

## Session Flow

### 1. Welcome (2 min)

**Actions:**
- Greet participant warmly
- Thank them for their time
- Read consent script verbatim
- Get verbal permission to record
- Start recording only after consent

**Script:**
> "Hi [Name], thanks so much for joining! Before we get started, I need to share some quick information about the session..."
>
> [Read consent script from consent-script.md]

### 2. Context Setting (1 min)

**Script:**
> "Great, we're recording now. Just a few quick things:
>
> - We're testing a guided interview feature that helps people map their product
> - Think out loud as you go - share what you're thinking, even if it seems obvious
> - There are no wrong answers - we're testing the product, not you
> - If something is confusing, that's valuable feedback for us
>
> Any questions before we begin?"

### 3. Task: Complete Interview (15 min)

**Actions:**
1. Paste Basesignal URL in Zoom chat
2. Ask participant to share their screen
3. Let them navigate naturally - don't guide unless stuck

**Script:**
> "I'm going to paste a link in the chat. When you're ready, please share your screen and start the interview. I'll mostly be quiet and observing, but feel free to ask questions if you get stuck."

**Observation Notes Template:**

| Time | Observation | Type |
|------|-------------|------|
| 0:30 | | Hesitation / Confusion / Engagement |
| 1:00 | | |
| 2:00 | | |
| ... | | |

**What to Watch For:**
- Where do they hesitate before answering?
- What questions do they re-read?
- Where do they seem confused?
- When do they seem engaged or confident?
- Do they abandon or skip anything?
- What do they say out loud?

**When to Intervene:**
- If truly stuck for >30 seconds, offer a gentle nudge
- If technical issue, help resolve quickly
- Otherwise, stay quiet and observe

### 4. Debrief (5 min)

**Core Questions:**
1. "What was that experience like for you?"
2. "Was anything confusing or frustrating?"
3. "What would make this better?"
4. "Any other thoughts?"

**Follow-up Probes:**
- "You mentioned [X] - can you tell me more about that?"
- "I noticed you paused at [point] - what were you thinking?"
- "On a scale of 1-10, how likely would you be to recommend this?"

**Script:**
> "Thanks for going through that. I have a few quick questions..."

### 5. Wrap-Up (1 min)

**Actions:**
- Thank participant sincerely
- Stop recording
- Mention any next steps or follow-up

**Script:**
> "That was really helpful - thank you! I'm going to stop the recording now. Is there anything else you'd like to share before we wrap up?"
>
> [Stop recording]
>
> "Thanks again for your time. I'll send a quick thank-you email. Have a great day!"

## Post-Session Checklist

Immediately after session ends:

- [ ] Verify recording saved (check Zoom cloud recordings)
- [ ] Add observation notes to shared document
- [ ] Note completion status:
  - Completed
  - Dropped (note where/why)
  - Technical issue (describe)
- [ ] Send thank-you email to participant
- [ ] Flag any critical insights for team

## Common Situations

### Participant Goes Off-Track

> "That's interesting - for now, let's focus on the interview. You can explore that after if you'd like."

### Technical Issues

> "No worries, these things happen. Let's try [solution]. If it keeps happening, we can reschedule."

### Participant Asks for Help

> "What would you try if I weren't here?"

(Only provide guidance if truly stuck)

### Participant Gets Frustrated

> "I can see this is frustrating - that's really valuable feedback. Would you like to take a break or continue?"

### Running Over Time

> "We're at the 15-minute mark. Would you like to wrap up here, or continue for a few more minutes?"

## Recording Management

| Delete recording | 30 days after analysis complete |
