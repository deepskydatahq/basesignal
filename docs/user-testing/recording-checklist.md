# Recording Setup Checklist

Setup guide for capturing user test sessions during H2: Interview Completion testing.

## Recording Method

**Primary:** Zoom (cloud recording)
**Backup:** Loom (for participants who prefer async)

### Why Zoom

- Live moderation allows follow-up questions
- Built-in cloud recording with transcript
- Participants already familiar with it
- Real-time observation of reactions

## Zoom Settings Configuration

### Step 1: Enable Cloud Recording

1. Go to Zoom web portal → Settings → Recording
2. Enable these options:
   - [x] Cloud recording
   - [x] Record active speaker with shared screen
   - [x] Audio transcript
   - [ ] Record thumbnails when sharing (disable for privacy)

### Step 2: Meeting-Level Settings

Before each test session:
- [x] Confirm recording is enabled for the meeting
- [x] Have participant share their screen (not yours)
- [x] Enable waiting room to control start time

## Quality Verification

### Test Recording Checklist

Before conducting real user tests, run a test recording:

- [ ] Start test Zoom meeting
- [ ] Enable screen share (simulating participant)
- [ ] Record for 2-3 minutes
- [ ] Navigate through Basesignal interview flow
- [ ] Speak clearly to test audio

### Quality Requirements

After test recording, verify:

| Aspect | Requirement | Verified |
|--------|-------------|----------|
| Video | Readable at 720p | [ ] |
| Audio | Clear and understandable | [ ] |
| Screen share | Text legible, UI visible | [ ] |
| Transcript | Accurately captures speech | [ ] |
| File size | <100MB for 15 minutes | [ ] |

### Recording Location

- Zoom cloud: Accessible via zoom.us → Recordings
- Download for backup if needed
- Keep for 30 days post-analysis, then delete

## Loom Backup Instructions

For participants who prefer async testing:

1. Send Loom download link: https://www.loom.com/download
2. Ask them to:
   - Record their screen and camera
   - Share screen with Basesignal open
   - Think out loud as they go through the interview
   - Send the Loom link when complete

## Pre-Session Equipment Check

Before each session:

- [ ] Stable internet connection
- [ ] Headphones plugged in (avoid echo)
- [ ] Quiet environment
- [ ] Zoom app updated
- [ ] Basesignal accessible and logged in

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Recording not starting | Check cloud recording is enabled in Zoom settings |
| Poor audio quality | Ask participant to use headphones |
| Screen share not visible | Have participant stop/restart share |
| Transcript not generating | Verify audio transcript is enabled in settings |
